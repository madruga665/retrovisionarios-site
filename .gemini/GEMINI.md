# Regras e Contexto do Projeto Retrovisionários

## Padrões de Desenvolvimento
- **Pull Requests:** Ao criar um Pull Request, você **DEVE OBRIGATORIAMENTE** ler o arquivo `.github/pull_request_template.md` e usar seu conteúdo como base para o corpo do PR. Preencha as seções do template (`:loudspeaker: O que está sendo feito?` e `:bulb: Como isso está sendo feito?`) com as informações relevantes das alterações realizadas. Nunca crie um PR com um corpo genérico se este template existir.
